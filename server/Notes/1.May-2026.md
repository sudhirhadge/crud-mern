## Login API Debugging

While testing the login route, received the error:

```txt id="ng3hm9"
Illegal arguments: number, string
```

### Root cause

`bcrypt.compare()` expects both arguments to be strings.

Incorrect:

```json id="ptgpk4"
{
  "password": 12345
}
```

Correct:

```json id="e85mbf"
{
  "password": "12345"
}
```

Also checked the Mongoose user schema and confirmed that password should be stored as a string.

# Middleware Order Matters

Mistakenly placed the error-handling middleware before routes while trying to group all middlewares together.

Learned that:

* Middleware execution order is important in Express.
* Error middleware should always be placed after all routes.

Correct structure:

```js id="87vhsy"
app.use(express.json());
app.use(cors());

app.use("/api/users", userRoutes);

app.use(errorMiddleware);
```

# Access Token vs Refresh Token

## Access Token

* Short-lived token (example: 15 minutes)
* Stored in React memory/state
* Sent in API requests as:

```txt id="t06yze"
Authorization: Bearer <token>
```

Used for accessing protected APIs.

---

## Refresh Token

* Long-lived token (example: 7 days)
* Stored in HTTP-only cookies
* Sent from backend using:

```js id="4tk2ln"
res.cookie("refreshToken", refreshToken)
```

and then:

```js id="j0czr8"
res.json({ accessToken })
```

Frontend JavaScript cannot directly access HTTP-only cookies, making them more secure.

---

# Refresh Token Flow

When the access token expires:

```txt id="1c1x5y"
401 Unauthorized
```

Frontend automatically calls:

```txt id="nuznfd"
POST /api/users/refresh
```

Backend:

* verifies refresh token
* extracts user information
* generates a new access token for that user

Logout should clear the refresh token cookie.

# Frontend Refresh Request

Frontend refresh request should include:

```js id="nys0zf"
axios.post(
  "/api/users/refresh",
  {},
  {
    withCredentials: true,
  }
);
```

Without `withCredentials: true`, cookies will not be sent.
