## Refresh Token Flow

### Issue 1 — `404 Not Found`

Frontend was making a `GET` request for refresh route.

Correct approach:

```txt
POST /api/users/refresh
```

### Why refresh should be POST

* Changes authentication state
* Generates new access tokens
* May rotate refresh tokens later
* Should not be cacheable like GET requests

---

### Issue 2 — `401 Unauthorized`

Reason:
Frontend login request was missing:

```js
withCredentials: true
```

Without credentials:

* browser does not store cookies from backend response
* refresh route receives no cookies

After adding credentials, refresh cookies started working correctly.

---

## Checking Cookies in Browser

Open DevTools:

```txt
Application
→ Cookies
→ localhost:5000
```

Check if:

```txt
refreshToken
```

exists.

If not:

* cookie was never stored in browser.
